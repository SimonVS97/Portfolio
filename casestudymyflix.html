Case study: myFlix Full-stack Project

Introduction

Summary

myFlix is a web-app and acts as a library for movies. Users have access to the stored movies and can access movie related information, such as information on the director, the genre and the movie itself. Moreover, users are able to create an account, update their personal data and create a list of favorite movies.

Objective

The goal of this project was to create a full-stack project that I can add to my software portfolio. Therefore, the problem I wanted I solve was to build the entire front-end and back-end on my own.

Approach

Back-end

The back-end consists of a RESTful API, that interacts with a NoSQL database. The API was created using Node.js and the Express framework and can be accessed using HTTP methods like GET or POST. The API is hosted on Netlify. The database was created with and is hosted on MongoDB. It also includes user authentication using JWT authorization and user authorization.

Tech Stack
See project on GitHub

Front-end

The front-end is a single-page, responsive application. It consists of a main view, that contains multiple sub-views. The design was created using react bootstrap. Several UI components enable the user to send requests to the server. Furthermore, the responses of the server are dynamically rendered in the respective views.

Conclusion

Experiences

In this project, I wanted to gain experience in full-stack development by building the entire client-side and server-side of a web-app on my own. My idea was to create myFlix, a movie library app.

First, I created the database that stores the data of the movies and the users. Then I set up the web server, that interacts with the database. Moving on to the front-end, I created different views and ensured that the views communicate properly with the web-server and display the data correctly. And with that, the project was finished.

All in all, creating myFlix was such a great learning experience. Building this web app taught me basic concepts such as how to create modern user interfaces while also introducing me to more advanced concepts like client-server-communication. myFlix really started my excitement about react development and encouraged me to keep learning.

Duration

Completing this project took me almost 3 weeks. The reason for this is that I needed spend some time to fully understand the different concepts of React.

Fallstudie: myFlix Full-Stack Projekt

Einführung

Zusammenfassung

myFlix ist eine Web-App und fungiert als Bibliothek für Filme. Die Nutzer haben Zugriff auf die gespeicherten Filme und können auf filmbezogene Informationen zugreifen, wie z. B. Informationen über den Regisseur, das Genre und den Film selbst. Außerdem können die Nutzer ein Konto anlegen, ihre persönlichen Daten aktualisieren und eine Liste ihrer Lieblingsfilme erstellen.

Ziel

Das Ziel dieses Projekts war es, ein Full-Stack-Projekt zu erstellen, das ich in mein Software-Portfolio aufnehmen kann. Das Problem, das ich lösen wollte, war daher, das gesamte Front-End und Back-End selbst zu erstellen.

Ansatz

Back-End

Das Back-End besteht aus einer RESTful-API, die mit einer NoSQL-Datenbank interagiert. Die API wurde mit Node.js und dem Express-Framework erstellt, Anfragen können mit HTTP-Methoden wie GET oder POST gesendet werden. Die API wird auf Netlify gehostet. Die Datenbank wurde mit MongoDB erstellt und wird dort gehostet. Die API Sie umfasst auch die Benutzerauthentifizierung mit JWT-Authentifizierung und Benutzerautorisierung.

Tech Stack
Sehen Sie sich das Projekt auf Github an

Front-End

Das Frontend ist eine single-page und responsive Web-Anwendung. Sie besteht aus einer Hauptansicht, die mehrere Unteransichten enthält. Das Design wurde mit React Bootstrap erstellt. Mehrere UI-Komponenten ermöglichen dem Benutzer das Senden von Anfragen an den Server. Darüber hinaus werden die Antworten des Servers dynamisch in den jeweiligen Ansichten gerendert.

Fazit

Erfahrungen

In diesem Projekt wollte ich Erfahrungen in der Full-Stack-Entwicklung sammeln, indem ich die Client- sowie Server-Seite einer Web-App selbst entwickle. Meine Idee war es, myFlix zu entwickeln, eine App für eine Filmbibliothek.

Zuerst habe ich die Datenbank erstellt, in der die Daten der Filme und der Benutzer*innen gespeichert werden. Dann richtete ich den Webserver ein, der mit der Datenbank interagiert. Für das Front-End habe ich verschiedene Ansichten erstellt und sichergestellt dass die Ansichten richtig mit dem Webserver kommunizieren und die Daten richtig anzeigen. Und damit war das war das Projekt abgeschlossen.

Alles in allem war die Entwicklung von myFlix eine großartige Lernerfahrung. Bei der Entwicklung dieser Webanwendung habe ich grundlegende Konzepte wie die Erstellung moderner Benutzeroberflächen, aber auch fortgeschrittene Konzepte wie Client-Server-Kommunikation praktisch umgesetzt. myFlix hat meine Begeisterung für die React-Entwicklung geweckt und mich ermutigt, weiter zu lernen.

Zeit

Für die Fertigstellung dieses Projekts habe ich fast 3 Wochen gebraucht. Der Grund dafür ist, dass ich einige Zeit brauchte, um die die verschiedenen Konzepte von React zu verstehen.
